# Guide for AI coding agents (Claude Code, ChatGPT, etc.)

Read this first. It tells you exactly where the site's text and design live, how to verify
a change, and how to ship it. The person instructing you is a **designer, not a developer** —
keep changes focused, explain what you changed in plain language, and always deploy the way
described under **"Shipping a change"** below.

## What this project is

A static single-page marketing site for **Neritix**. Stack: React 19 + Vite 7 +
Tailwind CSS v4. No backend — `npm run build` outputs static files that get uploaded to
our web host (Neoserv) by CI.

## Where the TEXT lives

All copy is hardcoded inline in the JSX of each component in `src/components/`. There is
**no CMS and no central text file** — edit the component that renders the section you see.

| On-page section | File |
| --- | --- |
| Top navigation bar + logo + menu links | `src/components/Nav.jsx` |
| Live data ticker (LIVE badge, vessel counts) | `src/components/LiveTicker.jsx` |

`src/components/`.

## Where the DESIGN lives

**Colors and fonts** are centralized in the `@theme` block at the top of
**`src/index.css`** (Tailwind v4 keeps its theme in CSS, not a JS config file):

```css
@theme {
  --color-bg: #0A0E14;            /* page background (deep navy) */
  --color-surface: #0F1824;       /* section surfaces */
  --color-card: #141F2E;          /* cards / modals */
  --color-accent-teal: #00C8B4;   /* primary accent */
  --color-accent-blue: #1E6FFF;   /* secondary accent */
  --color-text-primary: #E8EDF2;  /* main text */
  --color-text-muted: #7A8A9A;    /* secondary text */
  --color-border: #1E2D3D;        /* borders */
  --color-amber: #FFB347;         /* highlight / warning */

  --font-heading: 'Space Grotesk', sans-serif;
  --font-body: 'Inter', sans-serif;
  --font-mono: 'JetBrains Mono', monospace;
}
```

Changing a value here restyles the whole site. Fonts are loaded from Google Fonts in
`index.html` — if you switch a font family, update the `<link>` there too.

**Per-element styling** uses Tailwind utility classes inline in each component's JSX
(e.g. `text-accent-teal`, `bg-card`, `font-heading`). Animations use **Framer Motion**
(`motion.*`, `useInView`). A few chart/canvas colors are hardcoded hex values inside
`DataSelector.jsx`, `DataProducts.jsx`, and `BenthicCrossSection.jsx` — grep for the hex
if a chart color needs to match a theme change.

## Data

Demo content for the charts, map, and pricing lives in `public/data/*.json`, read via
helpers in `src/data/syntheticData.js`. It's all clearly-labeled **synthetic** data.

## Environment variables

Read from `.env.local` locally (git-ignored) and from GitHub Actions secrets in CI:
`VITE_MAPBOX_TOKEN` (map; required), and `VITE_EMAILJS_*` (contact form; currently unset →
form runs in harmless demo mode). Never commit `.env.local` or hardcode a token.

## Verify a change before shipping

```bash
npm install      # if node_modules is missing
npm run dev      # visually check at http://localhost:5173
npm run build    # MUST succeed — this is what CI runs
```

If `npm run build` fails, the deploy will fail. Fix it before pushing.

## Shipping a change  ← important

Deployment is fully automated. **Do not** build a zip or upload anything to the host.

```bash
git add -A
git commit -m "Describe the change in plain language"
git push origin main
```

Pushing to `main` triggers GitHub Actions, which builds the site and uploads `dist/` to
Neoserv over FTP. The change is live in ~1–2 minutes. You can confirm with `gh run watch`
or the repo's **Actions** tab (green ✓).

### Rules
- **Never edit the `dist/` folder** — it's generated by the build and overwritten every time.
- **Never commit `.env.local`** or paste real tokens/passwords into any tracked file.
- Keep the SPA routing file `public/.htaccess` intact — it makes deep links work on the host.
- After pushing, tell the user it's deploying and roughly when it'll be live.
